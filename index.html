📐 GeoTopo Web — Guía de uso

¿Qué es GeoTopo Web?

Aplicación web para generar topografía automática a partir de datos abiertos. Dibuja un área en el mapa, descarga elevaciones reales y exporta la información en múltiples formatos.

Creado por: Wladimir Campos · Algoritmos propios
www.jfsasesorias.org

Pasos para usar

1
Seleccionar área: Haz clic en el botón "📐 Seleccionar área en el mapa" y dibuja un rectángulo sobre la zona de interés.
2
Configurar capas: Elige las capas que necesitas (triangulación, curvas, caminos, agua, edificios, árboles, catastro).
3
Importar: Presiona "▶ Importar área seleccionada". El sistema descargará elevaciones automáticamente desde APIs públicas.
4
Exportar: Una vez procesado, usa los botones de exportación según el formato que necesites.

Formatos de exportación

  • DXF AutoCAD / Civil 3D — curvas, puntos, TIN, caminos
  • KML Google Earth — visualización 3D con altitudes
  • OBJ Blender / Rhino / SketchUp — malla 3D del terreno
  • STL Impresión 3D — malla para impresoras 3D
  • IFC BIM — Revit / ArchiCAD / FreeCAD

Capas disponibles

  • Triangulación Delaunay (TIN): Malla de triángulos que representa la superficie del terreno.
  • Curvas de nivel: Generadas automáticamente desde el TIN. Intervalo configurable (default 10m). Las maestras se dibujan cada 5× el intervalo.
  • Caminos y senderos: Desde OpenStreetMap. Con anchos reales según tipo de vía.
  • Cursos de agua: Ríos, arroyos, canales y drenes de OSM.
  • Deslindes / predios: Límites y uso de suelo de OSM.
  • Edificios: Construcciones de OSM con altura estimada.
  • Árboles: Nodos de árboles individuales de OSM.
  • Catastro / Parcelas: Polígonos catastrales de OSM.

Fuentes de elevación

El sistema detecta la región geográfica y elige automáticamente la mejor API disponible:

  • Auto (recomendado): selección inteligente por región
  • Francia → IGN RGE Alti — resolución 1m, la más precisa de Europa
  • Europa → EU-DEM 25m — Copernicus, sin autenticación
  • USA → USGS 3DEP — ~10m para territorio continental
  • Nueva Zelanda → LINZ 8m — vía opentopodata
  • Global → open-meteo.com — sin límite diario, muy estable
  • Fallback → opentopodata.org — SRTM 30m y ASTER 30m
  • Fallback final → open-elevation.com — alternativa SRTM

Si ves el indicador APIs OK en la cabecera, las elevaciones funcionarán correctamente. El dataset Auto es siempre la mejor opción.

Uso local (sin internet limitado)

Para mayor velocidad, sirve el archivo desde un servidor local:

python -m http.server 8080

Luego abre: localhost:8080/geotopo_web.html

Cobertura de curvas de nivel

Tras importar verás un indicador de cobertura. Si está bajo, presiona "⚡ Optimizar intervalo" para recalcular con un intervalo más adecuado al terreno. Los triángulos marcados como "planos" son inevitables cuando el DEM tiene resolución insuficiente.

Perfil Longitudinal

Genera perfiles topográficos sobre el terreno ya importado:

  • Importa el área con GeoTopo primero (TIN necesario)
  • Baja al panel → sección 📏 Perfil Longitudinal
  • Haz clic en ✏ Trazar línea y dibuja sobre el mapa. Doble-clic para terminar
  • Presiona ⚡ Generar perfil — usa el TIN local, sin internet
  • Exporta: DXF con cajetín AutoCAD, PNG, CSV o KML

Cota proyecto: ingresa una cota fija para ver el corte/relleno en el DXF (línea roja en el canvas).

KML — Altitud

El selector "Altitud KML" controla cómo se visualizan las entidades en Google Earth:

  • Al suelo (2D): Todo se pega al terreno de Google Earth
  • Absoluta (3D): Las entidades flotan a su elevación real. Los edificios siempre usan Absoluta para verse en 3D
  • Relativa al suelo: Elevación relativa sobre el DEM de Google
